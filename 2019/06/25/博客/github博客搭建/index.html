安装Node.js

Node.js 是一个基于 Chrome V8 引擎的 JavaScript运行环境。 Node.js 使用了一个事件驱动、非阻塞式 I/O 的模型。

下载64位9.11.1版本Node.js

默认安装后在命令行输入node -v和npm -v，如果出现版本号，那么就安装成功了。

本地配置

安装分布式版本控制工具Git，git下载地址。

之后下建立自己的博客文件夹，比如f/myBlog。在根目录下右键Git bash here打开git命令行界面。

定位到该目录下，输入npm install hexo-cli -g安装Hexo。会有几个报错，无视它就行。

然后输入npm install hexo --save继续安装。安装完后输入hexo -v验证是否安装成功。

输入hexo init初始化文件夹，接着输入npm install安装必备的组件。

输入hexo g生成静态网页，然后输入hexo s打开本地服务器，然后浏览器打开http://localhost:4000/，可以看到博客。(默认端口4000被占用的话可以hexo s -p8081指定8081端口)

在git bash 中输入设置git的信息

生成密钥SSH KEY

输入eval "$(ssh-agent -s)"，添加密钥到ssh-agent。

再输入ssh-add ~/.ssh/id_rsa，添加生成的SSH key到ssh-agent。

打开github，在头像下面点击settings，再点击SSH and GPG keys，新建一个SSH。

打开C:\Users\Administrator\.ssh\id_rsa.pub，注意是隐藏文件夹，将其中的内容复制到新建的SSH中。

输入ssh -T git@github.com，如果如下图所示，出现你的用户名，那就成功了。

设置github仓库

首先注册github账号，新建一个仓库，名字为[账户名].github.io。

发布一篇文章

通过hexo new post新建

然后输入hexo new post "article title"，新建一篇文章。

然后打开.\source\_posts的目录，下面多了一个文件夹和一个.md文件，一个用来存放你的图片等数据，另一个就是你的文章文件。（前提是在_config/yml中设置post_asset_folder: true用于支持本地图片上传和markdown图片路径转换）

编写完markdown文件后，

  1. 根目录下输入hexo g生成静态网页
  2. 然后输入hexo s -p8081后访问localhost:8081可以本地预览效果。
  3. 最后输入hexo d上传到github上，打开github.io主页就能看到发布的文章。

自己创建

可以自己新建一个markdown文件，放在_post的任意文件夹下，即可以自己新建目录用于分类放置各个类别的文章，不影响文章的发布，如果需要插入图片就需要自己新建同名文件夹，并在文件夹内放入需要的图片。

一些问题

插入图片

本地图片的markdown语法如下

Alt text可转换为html中标签中的alt属性，在图片无法显示的时候将代替显示对应的文本内容，小括号中可以输入绝对路径和相对路径，就需要插入![test](github博客搭建/1.png).hexo-assert-image将做路径转换，路径前输入./会造成路径转换错误，输入可选标题也会无法识别，虽然markdown可以识别。

插入图片

使用typora编辑器时可以设置 编辑->图片工具->设置图片根目录，这样插入时只需要输入图片名称如![test](1.png)。设置后可以发现在horizonal rules也就是文章最前面的设置中出现了typora-root-url: github博客搭建。

wechat图片

在浏览器中调试时可以发现在原主题中有对于icon_wechat.png（应该是微信图标）的请求并且我之前删掉了这个图片，导致请求不到，加载也很慢。于是在主题根目录下layout/layout.ejs中注释掉line24。

目录不显示

一是markdown中的toc标签不识别，生成了文本。二是博客主题中contents不显示，但是生成的html文件是正常的。目前没有找到原因。
